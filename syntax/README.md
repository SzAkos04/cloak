# Syntax Vault

Welcome to the **Syntax Vault** — a structured collection of notes on programming language syntax, grammar rules, code examples, and parsing theory.

This vault is designed to be used within [Obsidian](https://obsidian.md) for fast navigation, bidirectional linking, and long-term reference.

---

## Purpose

This vault serves as:

- A **reference guide** for syntax rules across different programming languages or a specific DSL.
- A **study and analysis hub** for grammar design, tokenization, and parsing strategies.
- A **living documentation system** for code patterns, syntax edge cases, and development notes.

---

## Structure

```plaintext
syntax-vault/
├── syntax/        # Core syntax patterns (expressions, types, functions, etc.)
├── grammar/       # Grammar notes (EBNF, parsing, lexer/parser notes)
├── examples/      # Code snippets, edge cases, broken syntax
├── meta/          # Tooling, changelog, conventions, etc.
```

---

## How to Use

- Use [[README#How to Use|wikilinks]] to jump between concepts.
- Syntax rules are primarily found in `syntax/`.
- Grammar theory and parser notes reside in `grammar/`.
- Use backlinks and tags (e.g. `#syntax`, `#grammar`) to connect related ideas.


---

## Conventions

- File naming: `lowercase-dash-separated.md`
- Code blocks use language identifiers for clarity (e.g. ` ```js `, ` ```python `).
- EBNF and other formal grammars are under `grammar/ebnf.md`.
