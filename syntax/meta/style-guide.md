# Style Guide

This document outlines the conventions for writing and formatting notes in the Syntax Vault.

## File Naming

- Use **lowercase** letters and **dash-separated** words.
- Examples: `let-statement.md`, `if-statement.md`, `grammar-rules.md`

## Formatting

- Use Markdown headers (`#`, `##`, `###`) to organize content hierarchically.
- Code blocks must specify the language or syntax for clarity.
- Use fenced code blocks with triple backticks (```` ``` ````).
- Inline code uses single backticks (`code`).

## Links and References

- Use [[style-guide#Links and References|wikilinks]] for internal references.
- Use descriptive link aliases, e.g., `[[syntax/functions|Functions]]`.

## Tags

- Use tags consistently for categorization (e.g., `#syntax`, `#grammar`).
- Tags go at the bottom of notes or in a dedicated metadata section.

## Writing Style

- Write in clear, concise language.
- Use examples liberally to illustrate syntax rules.
- Keep explanations focused on the topic of the note.
- Avoid personal opinions; focus on objective facts and standards.
